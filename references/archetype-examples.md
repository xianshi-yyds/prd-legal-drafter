# Archetype Examples

Use these worked examples to classify products against verified reference sources.

## Example 1: Direct AI Chat Product

- Input signals:
  - personal users log in directly
  - mobile verification
  - chat-first experience
  - no enterprise certification
- Best archetype:
  - `provider_to_c_mobile_login_ai_assistant`
- Closest verified source:
  - `千问`
- Drafting consequences:
  - use a clear C-end acceptance flow
  - emphasize input and output content handling
  - keep enterprise admin clauses out unless the PRD adds them

## Example 2: AI Meeting or Audio Understanding Product

- Input signals:
  - direct user use
  - audio or meeting transcript functions
  - likely Aliyun account inheritance
- Best archetype:
  - `provider_to_c_mobile_login_ai_media`
- Closest verified source:
  - `通义听悟`
- Drafting consequences:
  - check whether the legal and privacy layer is inherited from a platform account system
  - pay extra attention to audio, transcription, uploaded file, and meeting-record content

## Example 3: AI Image or Generative Media Product

- Input signals:
  - direct end-user creation
  - prompt-heavy
  - image or media output
- Best archetype:
  - `provider_to_c_mobile_login_ai_media_strong_product_terms`
- Closest verified source:
  - `通义万相`
- Drafting consequences:
  - add stronger acceptable-use and AI-output framing
  - inspect whether the privacy policy must discuss prompt, generated media, and uploaded reference files

## Example 4: AI Website Builder for SMB

- Input signals:
  - website generation
  - online customer service or content creation
  - workspace and order or renewal flows
  - individual or SMB buyers rather than strict enterprise certification
- Best archetype:
  - `provider_to_c_site_builder_with_workspace`
- Closest verified source:
  - `万小智AI建站`
- Drafting consequences:
  - combine direct-use consumer clauses with workspace and business-data handling
  - pay attention to renewal, domain, customer service, and business-contact data

## Example 5: Model or Agent Platform Console

- Input signals:
  - console or platform entrance
  - model APIs or app building
  - operator is a technical support or enablement layer
- Best archetype:
  - `supporter_console_platform`
- Closest verified source:
  - `阿里云百炼`
- Drafting consequences:
  - use platform-service style acceptance language
  - include API or integration access where supported by the PRD
  - do not overfit to consumer-chat wording

## Example 6: Enterprise Agent Construction Platform

- Input signals:
  - enterprise deployment
  - digital employee positioning
  - organization, admin, employee, workspace, or deployment emphasis
- Best archetype:
  - `supporter_enterprise_agent_platform`
- Closest verified source:
  - `AgentOne`
- Drafting consequences:
  - add admin and employee account structure
  - add enterprise procurement and enterprise data governance concerns
  - keep consumer-signup wording minimal

## Example 7: Tool-Calling or Claw Product

- Input signals:
  - skills
  - tool calling
  - file handling
  - webpage interaction
  - task execution
  - local or sandboxed execution risk
- Best archetype:
  - `work_ai_claw_high_privilege`
- Closest verified sources:
  - `JVS Claw`
  - `钉钉悟空`
  - `ArkClaw`
- Drafting consequences:
  - add stronger permission, execution-boundary, and risk-allocation clauses
  - make privacy coverage explicit for files, pages, third-party models, plugins, and execution traces
  - inspect whether the product behaves more like a worker or agent than a passive chat tool
