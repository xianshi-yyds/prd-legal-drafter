# 《灵犀纪要》产品 PRD（示例）

> 说明：本 PRD 为 Skill 测试用途的虚构样例，不对应任何真实对外上线产品。

## 1. 产品概述

- 产品名称：灵犀纪要
- 对外展示名称：灵犀纪要
- 运营主体：杭州灵犀云智科技有限公司
- 产品形态：Web 管理后台 + 员工移动端 H5
- 所属类型：企业 AI 会议纪要与知识整理工具
- 发布阶段：Beta 内测后商业化试运行

## 2. 产品定位

灵犀纪要面向企业客户提供会议内容整理服务，帮助企业在会议结束后自动生成纪要、待办事项、行动人、风险点及知识卡片，并支持企业管理员统一管理组织成员、会议空间和使用额度。

## 3. 目标用户

### 3.1 签约对象

- 产品主要面向企业客户销售，由企业作为签约主体采购服务。
- 企业签约后，可为本企业员工开通账号并分配使用权限。

### 3.2 实际使用人

- 企业管理员
- 企业员工账号

本产品不面向未成年人设计，也不以个人消费者为主要目标用户。

## 4. 核心场景

- 会后上传录音或会议文本，自动生成会议纪要
- 自动提取待办事项、负责人、截止日期
- 支持对纪要内容进行 AI 问答和再次总结
- 将纪要沉淀到企业知识库中，供企业成员检索
- 企业管理员统一管理成员、空间、使用量和权限

## 5. 账号与权限模型

### 5.1 登录方式

- 企业管理员由商务开通初始管理员账号，首次登录需通过手机号验证码验证并设置密码。
- 员工账号由企业管理员邀请开通。
- 员工登录支持手机号验证码登录。

### 5.2 角色类型

- 企业管理员：可创建空间、邀请/停用员工、查看企业级用量、管理订阅方案
- 员工账号：可在授权范围内上传会议资料、生成纪要、查看本人有权限访问的内容

### 5.3 账号绑定

- 员工账号与所属企业绑定。
- 员工离职或被管理员移除后，不再继续使用服务。

## 6. 商业化与结算

- 商业模式：SaaS 企业订阅
- 收费方式：按年购买席位包
- 试用：支持 14 天试用
- 续费方式：到期前由商务与企业确认后续费，不自动续费
- 开票：支持企业开票
- 退款规则：PRD 暂未明确，预计按合同约定执行

## 7. AI 与内容处理能力

- 支持上传音频文件、文本纪要、会议材料附件
- 系统会对上传内容进行转写、摘要、结构化提炼和问答
- 用户可输入 Prompt 对会议内容进行二次加工
- 生成结果包括纪要、待办、知识卡片、问答回复
- 企业成员可在企业内部共享纪要链接
- 默认不对外公开展示企业会议内容

## 8. 收集和处理的数据

### 8.1 账号注册与管理数据

- 手机号
- 验证码校验结果
- 姓名
- 头像（可选）
- 企业名称
- 部门、岗位（由企业管理员录入，可选）
- 账号角色、状态、所属企业

### 8.2 服务使用数据

- 登录时间
- 登录 IP
- 设备信息（设备型号、操作系统、浏览器信息）
- 操作日志
- 用量统计（上传次数、生成次数、存储量）

### 8.3 用户上传和生成内容

- 音频文件
- 会议文本
- 上传附件
- 用户输入的 Prompt
- 系统生成的转写结果、纪要、待办和知识卡片

### 8.4 商业化相关信息

- 企业联系人姓名
- 联系电话
- 联系邮箱
- 开票信息
- 合同编号和订单记录

### 8.5 客服与反馈

- 工单内容
- 联系方式
- 问题处理记录

## 9. 数据处理目的

- 完成账号注册、登录验证和身份识别
- 为企业管理员分配和管理成员权限
- 提供会议转写、纪要生成、问答与知识检索服务
- 统计服务使用量，用于席位管理、计费和运营分析
- 响应客户咨询、投诉与售后支持
- 进行故障排查和安全风控

## 10. 第三方能力与供应商

- 短信验证码服务：阿里云短信
- 对象存储：阿里云 OSS
- 大模型能力：通义千问 API

PRD 当前设计为：上传内容和 Prompt 会发送给大模型服务用于完成生成任务；当前产品方案中，不将企业上传内容用于本产品自训练。

## 11. 安全与风控

- 管理后台与服务接口使用 HTTPS 传输
- 企业管理员和员工按角色进行权限隔离
- 记录关键操作日志，用于审计和问题排查

## 12. 用户权利与客服路径

- 企业管理员可提交成员账号开通、停用、删除申请
- 企业客户可通过客户成功经理或客服邮箱发起数据导出、删除或其他支持请求
- 隐私与个人信息相关问题受理邮箱：privacy@lingxinotes.example
- 客服邮箱：support@lingxinotes.example

## 13. 当前未明确事项

- 个人信息和企业内容的具体保存期限暂未在 PRD 明确
- 是否存在跨境传输安排暂未明确
- 退款细则暂未在 PRD 明确
- 若接入更多第三方 SDK，需后续补充
